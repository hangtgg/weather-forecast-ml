# Contributing Guide

Cảm ơn bạn đã quan tâm đóng góp cho dự án!

## Quy tắc chung
- Giữ mã nguồn rõ ràng, có chú thích.
- Không commit dữ liệu nhạy cảm hoặc file quá lớn.
- Tuân thủ cấu trúc thư mục của dự án.

## Quy trình đóng góp
1. Fork repository.
2. Tạo nhánh mới:
   - `feature/<ten-tinh-nang>` cho tính năng mới
   - `fix/<ten-loi>` cho sửa lỗi
3. Commit với message rõ ràng.
4. Tạo Pull Request mô tả:
   - Mục đích thay đổi
   - Nội dung chính
   - Cách kiểm tra

## Báo lỗi / đề xuất
Vui lòng tạo Issue kèm:
- Mô tả lỗi/đề xuất
- Các bước tái hiện (nếu là lỗi)
- Log hoặc hình ảnh minh họa (nếu có)
